LLM Kernel Lab

Profile, benchmark, and optimize GPU kernels used in LLM inference. Compare Triton vs CUDA vs cuBLAS, generate roofline analyses, and catch performance regressions in CI.

Python 3.10+ CUDA / Triton / PyTorch NVIDIA SM 80+ Apache-2.0

Benchmark Harness

Reproducible benchmarks for attention, matmul, MLP, LayerNorm, and quantized GEMM kernels with predefined shapes for GPT-3, LLaMA, and Mistral.

Roofline Analysis

Automatic roofline plots showing arithmetic intensity vs achieved TFLOP/s with memory and compute roof lines for your GPU.

Kernel Comparison

Side-by-side comparison of Triton, handwritten CUDA, and cuBLAS/cuDNN implementations across problem shapes and GPUs.

Bottleneck Detection

Automatic classification — memory-bound, compute-bound, or latency-bound — with actionable optimization recommendations.

Nsight Integration

Wraps Nsight Compute CLI for detailed metrics: SM efficiency, occupancy, warp stall breakdown, L2 hit rates, and more.

CI Regression Guard

Built-in llm-kernel-lab ci command to detect performance regressions in GitHub Actions or GitLab CI pipelines.

Quick Start

# Install
pip install "llm-kernel-lab[all]"

# Detect GPU
llm-kernel-lab record-hw

# Run benchmarks
llm-kernel-lab bench \
  --op attention \
  --kernels flash_attn_v2_triton,sdpa_pytorch \
  --shapes llama2_7b_short,llama2_7b_long \
  --repetitions 100 \
  --profile-level timing \
  --output ./results/attn.json

# Generate report
llm-kernel-lab report \
  --input ./results/attn.json \
  --format markdown \
  --out ./reports/attn_report.md
